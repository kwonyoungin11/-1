# 오너 하루 운영 시나리오 (Cockpit)

실거래 전·후에도 같은 습관으로 본다.

## 아침 1분

1. 홈 연다  
2. **실거래 잠금**이 잠김인지 확인  
3. **긴급정지 ON**인지 확인 (기본 ON 권장 기간)  
4. 봇 상태 한 문장 읽기  
5. 빨간/노란 경고 있으면 “왜 막혔나” 카드 확인  

## 연결 직후 (Phase 2 이후)

1. 연결: 읽기 전용 정상?  
2. 계좌 마스킹된 요약이 보이는지  
3. 시장 배지(장중/장외)  

## 후보가 생겼을 때 (Phase 3+)

1. “주문 후보”는 **계획**이지 체결이 아님  
2. Risk Gate에서 통과/차단 이유 확인  
3. dry-run/paper 결과만 보고, 실주문 버튼은 없음  

## 이상할 때

1. 긴급정지 ON (이미면 유지)  
2. 기록(Audit) 최근 줄 읽기  
3. `docs/INCIDENT_RESPONSE.md`  
4. 토스 앱에서 잔고·주문 육안 확인  

## 절대 하지 말 것

- 키를 화면 캡처로 공유  
- “빨리 실거래 버튼 만들어줘”로 잠금 우회 요청  
- 차단 사유를 무시하고 재실행 반복
