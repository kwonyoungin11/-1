# 병렬 에이전트 개발 정책 (오너 확정)

**확정일:** 2026-07-09  
**오너 지시:** 모든 개발을 병렬 에이전트로 진행한다.

## 쉬운 설명

| 말 | 의미 |
|----|------|
| 병렬 에이전트 | 여러 AI 작업자가 **서로 다른 일**을 동시에 함 |
| 오케스트레이터 | 총괄 에이전트(나) — 목표 나누기, 결과 합치기, 안전 검사 |
| 독립 작업 | 같은 파일을 동시에 고치지 않는 단위 |

비유: 공사 현장에서 전기/배관/도장을 **동시에** 하되, 같은 벽을 두 팀이 동시에 뚫지 않게 구역을 나눔.

## 규칙

1. **2개 이상 독립 작업**이면 기본으로 병렬 에이전트를 띄운다.
2. 각 에이전트는 **담당 경로(파일 구역)** 만 수정한다.
3. 공통 파일(`AGENTS.md`, `.env.example`, `TradingBot.sln` 등) 변경은 **오케스트레이터만** 하거나 순차 병합한다.
4. 각 에이전트 종료 후: 요약 검토 → 충돌 확인 → `dev-loop.sh` → 커밋/병합.
5. **실주문·키 출력·live 기본값 완화 금지** (모든 에이전트 공통).
6. 작업 위치: git **worktree** (가능하면 에이전트별 브랜치/워크트리).
7. 투자 조언·수익 보장 문구 금지.

## 권장 스트림 예시 (현재 로드맵)

| 에이전트 | 구역 | 목표 |
|----------|------|------|
| A dry-run/audit | `Orders/`, `Domain` audit, tests | Phase 4 dry-run 원장·감사 |
| B observability | `Observability/` only | 구조화 감사 로그 모듈 |
| C cockpit UI model | `Ui/`, `docs/cockpit` | 화면 바인딩 모델 확장 |
| (옵션) D risk expand | `Risk/`, Risk.Tests | 리스크 규칙 추가 |
